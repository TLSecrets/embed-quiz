# 📘 嵌入式应用开发 — 智能刷题平台
**独立纯静态嵌入式刷题网站**，原生 HTML + CSS + ES6 JavaScript，无后端、无数据库、无任何第三方框架库。双击 `index.html` 即可本地运行，直接部署 GitHub Pages 即可在线访问。
> 📚 题库：167 道嵌入式应用开发客观题（单选 99 道 / 判断 42 道 / 填空 15 道 / 多选 11 道）
---
## 🔧 功能清单
### 1. 多种刷题模式
| 模式 | 说明 |
|---|---|
| **常规刷题** | 答对延时自动下一题，答错停住，实时显示对错，逐题推进 |
| **背题浏览** | 直接展示标准答案，不做判分，适合考前快速过题 |
| **模拟考试** | 答完所有题目后统一判分，无自动跳转，无中途反馈，模拟真实考场 |
### 2. 题型分类筛选
- 顶部按钮一键切换：**全部** / **单选** / **多选** / **填空** / **判断**
- 支持 URL 参数 `?type=radio` 直接进入指定题型，方便分享链接
- **严格过滤**：切换题型后仅展示对应题型题目，不会混入其他题型
- 恢复进度时自动检查保存的题型与当前筛选是否匹配，不匹配则重建题目池
### 3. 随机打乱 ABCD 选项（双向映射）
- 每次练习自动对选项做随机打乱，避免机械记忆位置
- 内置 **双向映射表**（`shuffledCache`），用户看到的选项与实际答案一一对应
- 答案展示、判分均使用映射后的位置，**杜绝答案错位**
- 恢复进度时同步恢复映射表，刷新页面后选项位置不变
### 4. 计分规则
- **单选 / 判断**：答对得分，答错/未答不得分
- **多选**：必须全部选中正确答案才得分，部分正确不得分
- **填空**：自动忽略空格与大小写差异，模糊匹配关键词即判定正确
- 练习报告**单独统计未答题**，与正确/错误分开列示
### 5. 自动跳转规则
| 场景 | 行为 |
|---|---|
| 常规刷题 — 答对 | 延时 600ms 自动跳下一题 |
| 常规刷题 — 答错 | 停留在当前题目，手动点击下一题 |
| 模拟考试 | 无自动跳转，手动完成全部题目后统一交卷 |
| 背题浏览 | 手动翻阅，无判分、无自动跳转 |
### 6. 自定义数量 + 重做练习
- 首页可**自定义单次练习题目数量**（默认 20 题，支持 5–167）
- 刷题页面右上角 **「重做本次练习」** 按钮，弹窗确认后清空本次记录、重新随机抽题
- 退出刷题页面时**自动清空本次练习临时作答痕迹**（`beforeunload` 事件触发），不留历史误选
- 错题训练页面不会自动回填历史错误选项，每次进入都是全新作答
### 7. 练习报告页
- 答完最后一题**自动跳转**练习报告页（`report.html`）
- 环形得分图（Canvas 绘制）+ 详细统计：正确数、错误数、未答数、正确率
- UI 风格统一，字号/颜色/布局与主页面一致，无双层分数重叠问题
### 8. 错题本多维管理
- 所有答错的题目自动收集到错题本（`wrong.html`）
- 支持**按错误次数升降序、按错误率升降序**排序
- 每道错题展示**各选项被选择次数**，精准定位薄弱知识点
- **自定义标签**：给错题打标签分类
- **收藏**：标记重点题目
- **个人笔记**：每道题可写私人笔记，localStorage 持久保存
- 错题本内可直接**重做全部错题**、**一键清空错题本**
### 9. 全局设置
- **深色 / 浅色模式**：一键切换，偏好持久化存储，刷新不丢失
- **键盘快捷键**：
  - `1` `2` `3` `4` — 快速选择 ABCD 选项
  - `Enter` — 确认/下一题
  - `←` `→` — 上一题/下一题
- 输入框内自动禁用快捷键，不影响正常打字
### 10. 答题卡 + 本地持久化
- 底部**答题卡缩略方格**，清晰显示每题作答状态（✅ 正确 / ❌ 错误 / ⬜ 未答）
- 点击方格可**快速跳转**到对应题号
- 全部数据（错题、笔记、收藏、统计、设置偏好）均通过 `localStorage` 持久化存储
### 11. 底层优化与修复项
| # | 优化项 | 说明 |
|---|---|---|
| ① | 随机抽题去重 + 索引越界防护 | `Picker.build` 内置去重逻辑，`qIndex` 始终限制在 `qList.length - 1`，不会出现 `undefined` 空白题 |
| ② | 命名空间 + 闭包隔离 | `App`、`Storage`、`State`、`Hotkey`、`Timer` 等模块均挂载独立命名空间下，无全局变量互相污染 |
| ③ | CSS 唯一前缀隔离 | 所有样式使用 `.eq-` / `#eq-` / `[data-eq-]` 前缀，不污染宿主页面样式 |
| ④ | 浏览器历史安全 | 无 `pushState` / `replaceState` / `hash` 篡改，不使用 `document.write` |
| ⑤ | 全链路异常捕获 | 题库加载失败显示友好提示，关键渲染路径 `try-catch`，不会页面白屏 |
| ⑥ | 统一 I18N 文字配置 | 所有中文文案集中在 `i18n.js`，修复 `I18N` 未定义报错；代码按功能拆分为 11 个 JS 模块文件 |
| ⑦ | 定时器统一销毁 | `Timer` 模块统一管理 `setInterval` / `setTimeout`，页面离开时全量 `clear`，无内存泄漏 |
| ⑧ | 纯原生 JS，无第三方依赖 | 无 jQuery / Vue / React 依赖，资源使用相对路径，无跨域 CORS 问题 |
| ⑨ | 移动端弹性响应 | `mobile.css` 适配，窄容器下字体/按钮/选项正常缩放，不溢出不重叠 |
| ⑩ | DOM 批渲染 | 题型切换时局部更新 DOM，不做整页 `innerHTML` 重绘，避免页面卡顿闪烁 |

## 📁 项目目录结构
```
embed-quiz/
├── index.html            # 首页 — 入口、统计总览、题型入口、自定义数量
├── exam.html             # 刷题主页面 — 核心答题页
├── wrong.html            # 错题本 — 错题管理、排序、标签、笔记、重做
├── report.html           # 练习报告 — 环形得分图、详细统计
├── css/
│   ├── style.css         # 主样式表（~27KB），.eq- 前缀隔离
│   └── mobile.css        # 移动端响应式样式（~2.4KB）
├── js/
│   ├── question.js       # 题库数据 — 167 道题目结构化数组
│   ├── app.js            # 应用入口 — 初始化、模式配置、页面路由
│   ├── page.js           # 核心刷题逻辑 — 切题、判分、进度、报告跳转
│   ├── common.js         # 通用工具 — 题型过滤、选项比对、答案格式化
│   ├── renderer.js       # DOM 渲染 — 题目卡片、选项列表、答题卡
│   ├── storage.js        # 持久化层 — localStorage 读写封装
│   ├── state.js          # 状态管理 — 全局状态读写
│   ├── picker.js         # 随机抽题 — Picker.build 去重抽取逻辑
│   ├── i18n.js           # 国际化 — 所有中文文案集中管理
│   ├── timer.js          # 定时器 — 统一创建/销毁，防泄漏
│   ├── darkmode.js       # 深色模式 — 切换 + 持久化
│   └── hotkey.js         # 键盘快捷键 — 1/2/3/4 + Enter + 方向键
└── images/
    └── *.png             # 题目配图（3 张），按需替换
```

## 🚀 本地运行
1. **克隆仓库**
   ```bash
   git clone https://github.com/YOUR_USERNAME/embed-quiz.git
   cd embed-quiz
   ```
2. **直接打开** `index.html`（双击或用浏览器打开即可）
   — 无需启动服务器、无需安装任何依赖。

## 🌐 GitHub Pages 部署
1. 在 GitHub 上创建新仓库（如 `embed-quiz`）
2. 将本目录所有文件推送到仓库：
   ```bash
   git init
   git add -A
   git commit -m "Initial commit — 嵌入式应用开发刷题平台"
   git remote add origin https://github.com/YOUR_USERNAME/embed-quiz.git
   git branch -M main
   git push -u origin main
   ```
3. 进入仓库 **Settings → Pages**
4. **Source** 选择 `Deploy from a branch`
5. **Branch** 选择 `main`，目录选 `/ (root)`，点击 **Save**
6. 等待 1–2 分钟，访问 `https://YOUR_USERNAME.github.io/embed-quiz/` 即可在线使用

## 🛠 维护教程
### 修改 / 新增题库题目
编辑 `js/question.js`，找到 `const questionList = [` 数组，按以下格式添加或修改：
```javascript
{
  id: 168,                      // 题号（唯一，递增）
  type: 'radio',                // radio | checkbox | fill | judge
  title: '题干内容……',
  img: 'images/new_image.png',  // 无图片填空字符串 ''
  options: ['选项A', '选项B', '选项C', '选项D'],
  answer: 'A',                  // 单选: 'A' | 多选: 'A,C' | 填空: '答案' | 判断: 'A'(对)或'B'(错)
  tags: [],                     // 标签数组，可选
  collect: false,               // 初始是否收藏
  note: ''                      // 初始笔记
},
```
**字段说明**：
| 字段 | `radio`(单选) | `checkbox`(多选) | `fill`(填空) | `judge`(判断) |
|---|---|---|---|---|
| `options` | `['A', 'B', 'C', 'D']` | 同上 | `[]`(空数组) | `[]`(空数组) |
| `answer` | `'A'` | `'A,C'`(逗号分隔) | `'答案文本'` | `'A'`=对, `'B'`=错 |
### 替换题目图片
1. 将图片文件放入 `images/` 目录
2. 修改 `js/question.js` 中对应题目的 `img` 字段为 `'images/文件名.png'`
3. 无图片的题目 `img` 字段保持空字符串 `''`
### 各功能使用说明
**随机打乱选项**：每次进入练习自动随机打乱，同一题在不同练习中选项位置不同，防止机械记忆。选项映射表随进度一起保存，恢复后位置不变。
**重做本次练习**：刷题页面右上角点击「重做练习」，弹窗确认后清空当前所有作答记录，重新随机抽取题目。
**错题训练**：在「错题本」页面点击「重做全部错题」进入错题专项训练。注意：错题训练每次进入都是全新作答，不会回填历史错误选项。
**深色模式**：首页或刷题页点击 🌙/☀️ 按钮切换，偏好自动保存到 localStorage。
**键盘快捷键**：
| 按键 | 功能 |
|---|---|
| `1` `2` `3` `4` | 选择 A / B / C / D |
| `Enter` | 确认答案 / 跳下一题 |
| `←` `→` | 上一题 / 下一题 |

## ❓ 常见问题排查
| 问题 | 原因 | 解决方案 |
|---|---|---|
| **题目加载空白** | 题库文件 `js/question.js` 加载失败 | 检查浏览器控制台 Network 面板，确认文件路径正确、无 404；确保 HTML 中 `<script src="js/question.js">` 引用顺序在最前 |
| **题型分类筛选后为空** | 题库中无该题型数据 | 确认 `question.js` 中题目 `type` 字段正确：`'radio'` / `'checkbox'` / `'fill'` / `'judge'`（注意小写），共 167 道 |
| **控制台报 I18N 未定义** | JS 文件加载顺序错误 | `i18n.js` 必须在 `page.js` / `renderer.js` 之前加载；检查 HTML 中 `<script>` 标签顺序 |
| **筛选题型混入其他题型** | 过滤逻辑未生效 | 确保 `Picker.build` 传入了 `filterType` 参数；`rebuildQList` 调用时传递了当前筛选类型；刷新后清除 localStorage 中的旧进度重新测试 |
| **随机选项答案错位** | 选项打乱后未同步映射 | 检查 `shuffledCache` 是否正确保存和恢复；确保判分时使用 `optionMap` 做反向映射；刷新后若映射丢失，清除 localStorage 重新练习 |
| **页面样式冲突** | 宿主页面 CSS 与刷题样式互相覆盖 | 本平台全部样式使用 `.eq-` 前缀，理论上不冲突；如仍冲突，请检查宿主页面是否有 `!important` 规则覆盖 |
| **移动端布局错乱** | 窄容器下元素溢出 | `mobile.css` 已做响应式适配，确认该文件正确加载；如仍有问题，检查宿主页面的 viewport meta 标签 |
