## 🎮 功能详解
### 四种刷题模式
| 模式 | 说明 |
|------|------|
| **常规刷题** | 选完立刻判定对错，显示解析，可做笔记标签 |
| **背题浏览** | 直接展示题目+答案，快速过知识点 |
| **模拟考试** | 关闭即时反馈，全部完成统一出分+报告 |
| **错题训练** | 只刷历史错题，clean start 重新作答 |
| **🔄 重做本次练习** | 一键清空当前练习全部作答，从头开始 |
### 错题本
- **多维度筛选**：按题型、标签、收藏状态筛选
- **多种排序**：错误次数↓↑ / 错误率↓↑ / 最近错误↓↑
- **详情弹窗**：查看单题错误次数、各选项被选分布、个人笔记
- **错题训练**：直接从错题本一键进入专项训练
### v6.0 新增行为
- **离开页面自动清理**：关闭/刷新/返回首页时，自动清除本次练习的临时作答记录（错题库不受影响）
- **最后一题自动报告**：非考试模式下，刷完最后一题自动生成报告并跳转，无需手动操作
- **错题模式 clean start**：进入错题训练时选项全部未选中，像第一次做题一样
### 键盘快捷键
| 快捷键 | 功能 |
|--------|------|
| `1` `2` `3` `4` | 快速选择 A / B / C / D |
| `←` `→` | 上一题 / 下一题 |
| `Enter` | 跳转下一题 |
### 数据存储说明
所有数据存储在浏览器 localStorage 中：
| Key | 内容 |
|-----|------|
| `eq_a_*` | 每道题作答记录 |
| `eq_o_*` | 各选项选择次数统计 |
| `eq_wrong_list` | 错题ID列表 |
| `eq_wrong_detail` | 错题详细统计（错误次数/错误率/最近错误时间） |
| `eq_progress` | 当前练习进度（题型/模式/题号） |
| `eq_global_stats` | 全局统计（练习次数/总做题量） |
| `eq_user_pref` | 用户偏好（深色模式/快捷键/选项随机/自动跳转延迟） |
| `eq_tags_custom` | 自定义标签库 |
| `eq_q_meta` | 题目元数据（标签/收藏/笔记） |
| `eq_last_report` | 最近一次练习报告（供报告页读取） |

## 🔧 题库维护
### 新增题目
编辑 `js/question.js`，在 `questionList` 数组末尾追加：
```javascript
{
  id: 168,                     // 唯一递增题号
  type: 'radio',               // radio/checkbox/fill/judge
  title: '完整题干文本',
  img: 'images/xxx.png',       // 无图片填空字符串 ""
  options: ['选项A', '选项B', '选项C', '选项D'],
  answer: 'A'                  // 多选逗号分隔，填空顿号分隔，判断A=对B=错
}
```
### 替换图片
1. 将图片放入 `images/` 目录
2. 修改对应题目的 `img` 字段：`"images/你的文件名.png"`
3. 图片建议宽度 ≤ 800px，格式PNG/JPG

## ❓ 常见问题
**Q: 换浏览器后数据还在吗？**
A: 不在。数据绑定浏览器本地存储，不同浏览器不互通。

**Q: 清除浏览器缓存会丢失数据吗？**
A: 会。建议定期通过首页"全局设置"手动导出复习进度备忘。

**Q: 填空题怎么判对？**
A: 模糊匹配，包含参考答案关键词即判对。多空用顿号或逗号分隔。

**Q: 判断题的A/B对应什么？**
A: A = 对（正确），B = 错（错误）。

**Q: 错题训练时为什么看不到之前的答案？**
A: v6.0 起错题训练采用 clean start 策略，不再预填历史答案，强制重新独立作答以检验真实掌握程度。

**Q: 刷完一套题会自动跳转报告吗？**

 A: 非考试模式会自动跳转 report.html 展示本次成绩。考试模式需手动交卷。
